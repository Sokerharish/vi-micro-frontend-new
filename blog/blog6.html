ASYSTR 3C Cobot — Pick & Place Automation with ROS2 MoveIt

by Support Team  /  April 2026

Introduction

The ASYSTR 3C is a 6-axis collaborative robot with ±0.02 mm repeatability and built-in force-limiting safety. In this tutorial we program a complete pick-and-place task using ROS2 MoveIt — defining target poses, planning collision-free trajectories, and executing gripper control with force feedback for safe collaborative operation.

Working Principle

MoveIt uses the OMPL (Open Motion Planning Library) to plan collision-free joint trajectories from the current pose to a goal Cartesian pose. The ASYSTR 3C's built-in force sensor monitors contact forces — if the force exceeds the safety threshold during a pick operation, the cobot stops automatically without human intervention.

  • Forward Kinematics — Given joint angles → compute end-effector position
  • Inverse Kinematics — Given target position → solve for joint angles (KDL / TracIK)
  • OMPL Planner — RRT* or BiTRRT finds collision-free path in joint space
  • Trajectory Execution — Joint trajectory controller sends commands to each servo
  • Force Limiting — Real-time force monitoring stops motion if threshold exceeded

Step 1 — Setup MoveIt for ASYSTR 3C

Requires ROS2 Humble + MoveIt2 installed. The ASYSTR 3C URDF and MoveIt config package must be sourced from your workspace.
Terminal — Launch MoveIt with ASYSTR 3C
# Source ROS2 and workspace
source /opt/ros/humble/setup.bash
source ~/ros2_ws/install/setup.bash

# Launch ASYSTR 3C MoveIt demo
ros2 launch asystr3c_moveit_config demo.launch.py

# In another terminal — check robot state
ros2 topic echo /joint_states

Step 2 — Define Pick and Place Poses

pick_place.py — MoveIt Python API
import rclpy
from rclpy.node import Node
from moveit2 import MoveIt2
from geometry_msgs.msg import Pose
import time

class PickPlaceNode(Node):
    def __init__(self):
        super().__init__("pick_place_node")
        self.moveit = MoveIt2(
            node=self,
            joint_names=["joint1", "joint2", "joint3",
                         "joint4", "joint5", "joint6"],
            base_link_name="base_link",
            end_effector_name="tool0",
            group_name="asystr3c_arm"
        )

    def move_to_pose(self, x, y, z, qx=0, qy=0,
                          qz=0, qw=1):
        pose = Pose()
        pose.position.x    = x
        pose.position.y    = y
        pose.position.z    = z
        pose.orientation.x = qx
        pose.orientation.y = qy
        pose.orientation.z = qz
        pose.orientation.w = qw

        self.moveit.move_to_pose(pose)
        self.moveit.execute()
        print(f"Moved to: x={x}, y={y}, z={z}")

    def run_pick_place(self):
        # 1. Move to home position
        self.move_to_pose(0.0, 0.0, 0.5)
        time.sleep(1.0)

        # 2. Move above pick location
        self.move_to_pose(0.3, 0.0, 0.3)
        time.sleep(0.5)

        # 3. Lower to pick position
        self.move_to_pose(0.3, 0.0, 0.1)
        print("Closing gripper — picking object")
        time.sleep(1.0)

        # 4. Lift object
        self.move_to_pose(0.3, 0.0, 0.4)
        time.sleep(0.5)

        # 5. Move to place location
        self.move_to_pose(-0.3, 0.2, 0.4)
        time.sleep(0.5)

        # 6. Lower to place position
        self.move_to_pose(-0.3, 0.2, 0.1)
        print("Opening gripper — placing object")
        time.sleep(1.0)

        # 7. Return to home
        self.move_to_pose(0.0, 0.0, 0.5)
        print("Pick and place cycle complete!")

def main():
    rclpy.init()
    node = PickPlaceNode()
    node.run_pick_place()
    rclpy.shutdown()

if __name__ == "__main__":
    main()

Step 3 — Add Collision Objects

add_collision.py — Define Workspace Boundaries
from moveit2 import MoveIt2, CollisionObject
from shape_msgs.msg import SolidPrimitive

# Add table as collision object
table = CollisionObject()
table.id = "table"
table.primitive_poses[0].position.z = -0.05

box = SolidPrimitive()
box.type       = SolidPrimitive.BOX
box.dimensions = [1.0, 1.0, 0.1]  # 1m × 1m × 10cm

moveit.add_collision_object(table)
print("Table collision object added to planning scene")

Force Feedback Safety

Collaborative Safety: The ASYSTR 3C monitors contact forces in real time. If force exceeds the configured threshold (default 150N), the cobot stops immediately. Always set appropriate force limits before running near humans.
Terminal — Monitor Force Feedback Topic
# Monitor end-effector wrench (force/torque)
ros2 topic echo /asystr3c/wrench

# Expected output during pick:
wrench:
  force:
    x: 0.12
    y: -0.08
    z: 8.45    # Weight of object in Newtons
  torque:
    x: 0.001
    y: 0.002
    z: 0.0
A successful pick-and-place cycle takes approximately 8–12 seconds depending on the path length and velocity scaling. Start with velocity_scaling=0.3 and acceleration_scaling=0.3, then increase gradually after verifying trajectory correctness in RViz2.
