Getting Started with micro-ROS on ESP32
for Autonomous Navigation

June 2026 Vi Microsystems Team ROS2, ESP32, micro-ROS
Home› Blog› micro-ROS on ESP32

Introduction

micro-ROS is a lightweight version of ROS2 designed specifically for microcontrollers. Unlike a standard ROS2 setup that runs on Linux-capable hardware like a Raspberry Pi, micro-ROS allows the ESP32 to participate directly in a ROS2 network — publishing sensor data, receiving velocity commands, and running real-time hardware control.

In this guide, you will flash micro-ROS firmware on the ESP32, set up the XRCE-DDS Agent on a PC or Jetson, and verify communication using ROS2 command-line tools and RViz2.

Working Principle

The ESP32 runs a micro-ROS node that uses the Micro XRCE-DDS middleware to communicate with a micro-ROS Agent running on the host (PC or Jetson Orin). The Agent acts as a bridge between the embedded node and the full ROS2 DDS network.

  • ESP32 reads MPU6050 IMU data over I2C at 100 Hz
  • Encoder ticks are counted for wheel odometry calculation
  • Motor PWM commands are received from the ROS2 /cmd_vel topic
  • Battery voltage is read via ADC and published as a ROS2 topic
  • Ultrasonic sensor distances are published for obstacle avoidance

Step 1 — Install micro-ROS on ESP32 (Arduino IDE)

Requires Arduino IDE 2.x and ESP32 board package installed. Tested with micro-ROS Arduino library v4.3.0
Arduino IDE — Board Manager URL
https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

Install the micro-ROS Arduino library from the Library Manager — search for micro_ros_arduino and install version 4.3.0 or later.

Step 2 — ESP32 Firmware Code

main.ino — micro-ROS ESP32 Publisher
#include <micro_ros_arduino.h>
#include <rcl/rcl.h>
#include <sensor_msgs/msg/imu.h>
#include <geometry_msgs/msg/twist.h>

// micro-ROS handles
rcl_publisher_t     imu_publisher;
rcl_subscription_t  cmd_sub;
rclc_executor_t     executor;
rclc_support_t      support;
rcl_allocator_t     allocator;
rcl_node_t          node;

sensor_msgs__msg__Imu       imu_msg;
geometry_msgs__msg__Twist   cmd_msg;

void cmd_vel_callback(const void * msgin) {
  const geometry_msgs__msg__Twist * msg =
    (const geometry_msgs__msg__Twist *)msgin;
  // Set motor PWM from linear.x and angular.z
  setMotors(msg->linear.x, msg->angular.z);
}

void setup() {
  set_microros_transports();         // USB Serial transport
  allocator = rcl_get_default_allocator();
  rclc_support_init(&support, 0, NULL, &allocator);
  rclc_node_init_default(&node, "esp32_bot", "", &support);

  // Publisher — IMU data
  rclc_publisher_init_default(&imu_publisher, &node,
    ROSIDL_GET_MSG_TYPE_SUPPORT(sensor_msgs, msg, Imu),
    "/imu/data");

  // Subscriber — velocity commands
  rclc_subscription_init_default(&cmd_sub, &node,
    ROSIDL_GET_MSG_TYPE_SUPPORT(geometry_msgs, msg, Twist),
    "/cmd_vel");

  rclc_executor_init(&executor, &support.context, 2, &allocator);
  rclc_executor_add_subscription(&executor, &cmd_sub,
    &cmd_msg, &cmd_vel_callback, ON_NEW_DATA);
}

void loop() {
  readIMU(&imu_msg);               // Read MPU6050
  rcl_publish(&imu_publisher, &imu_msg, NULL);
  rclc_executor_spin_some(&executor, RCL_MS_TO_NS(10));
  delay(10);
}

Step 3 — Start micro-ROS Agent on PC / Jetson

Terminal — Run Agent (Docker)
# Pull and run micro-ROS agent via Docker
docker run -it --rm -v /dev:/dev --privileged \
  microros/micro-ros-agent:humble \
  serial --dev /dev/ttyUSB0 -v4

# OR build from source on Jetson:
source /opt/ros/humble/setup.bash
ros2 run micro_ros_agent micro_ros_agent \
  serial --dev /dev/ttyUSB0

Step 4 — Verify ROS2 Topics

Terminal — ROS2 Topic Check
# List all topics published by ESP32
ros2 topic list

# Expected output:
/imu/data
/battery/voltage
/ultrasonic/front
/ultrasonic/rear
/odom

# Echo IMU data
ros2 topic echo /imu/data

# Send velocity command to robot
ros2 topic pub /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.2}, angular: {z: 0.1}}"

Step 5 — Visualize in RViz2

Launch RViz2 and add the following displays:

  • TF — view robot transform tree
  • Odometry — subscribe to /odom
  • Imu — subscribe to /imu/data
  • LaserScan — if LiDAR is connected (for UNO Q / Orin)
Terminal — Launch RViz2
rviz2

Expected Results

When everything is working correctly, you should see IMU orientation arrows moving in RViz2 as you tilt the robot, and the robot chassis moving when you publish to /cmd_vel.

Conclusion

You have successfully set up micro-ROS on the ESP32, connected it to a ROS2 host, and verified sensor data and motor control over the ROS2 network. This forms the foundation for building a fully autonomous robot using the MR Robo ESP32 platform with Nav2, SLAM, and sensor fusion in future tutorials.
