Build an Offline AI ChatBot on Arduino UNO Q using SmolLM2 + llama.cpp + Flask

by Support Team  /  June 2026

Introduction

In this project you will run a complete Large Language Model (LLM) entirely offline on the Arduino UNO Q single-board computer — no internet, no cloud API, no OpenAI subscription needed. The entire inference stack runs locally using llama.cpp as the inference engine, SmolLM2-360M as the model, and Flask as the web server frontend.

Working Principle

The user types a message in the browser. The Flask backend receives the POST request and forwards it to the llama.cpp server running on localhost port 8080. The llama.cpp server passes the prompt to the SmolLM2 model for inference. The model generates a reply token-by-token and returns it. Flask sends the reply back to the browser UI.

Step 1 — Install Dependencies

Terminal — System Packages
sudo apt update && sudo apt upgrade -y
sudo apt install -y build-essential cmake git \
  libopenblas-dev libssl-dev python3 python3-venv

Step 2 — Build llama.cpp

Terminal — Clone and Build
cd ~
git clone https://github.com/ggerganov/llama.cpp.git
cd llama.cpp && mkdir build && cd build
cmake .. -DGGML_BLAS=ON \
         -DGGML_BLAS_VENDOR=OpenBLAS \
         -DLLAMA_OPENSSL=ON
cmake --build . -j2

Step 3 — Download SmolLM2 Model

Terminal — Download GGUF Model
cd ~/llama.cpp/build/bin
./llama-cli -hf \
  HuggingFaceTB/SmolLM2-360M-Instruct-GGUF:q4_k_m

Step 4 — Start llama.cpp Inference Server

Important: Keep this terminal open the entire time. Do NOT press Ctrl+C.
Terminal 1 — llama-server
./llama-server \
  -hf HuggingFaceTB/SmolLM2-360M-Instruct-GGUF:latest \
  -t 2 -c 256 --port 8080 --no-mmap

# Success: server is listening on http://127.0.0.1:8080

Step 5 — Flask Web App (app.py)

app.py — Flask Backend
from flask import Flask, request, jsonify, render_template_string
import requests

app = Flask(__name__)
LLM_URL = "http://localhost:8080/v1/chat/completions"

def ask_llm(prompt):
    payload = {
        "model": "smollm2",
        "messages": [{"role": "user", "content": prompt}],
        "max_tokens": 200
    }
    try:
        response = requests.post(LLM_URL, json=payload)
        return response.json()["choices"][0]["message"]["content"]
    except:
        return "Error: LLM server not running."

@app.route("/")
def index(): return render_template_string(HTML)

@app.route("/chat", methods=["POST"])
def chat():
    user_msg = request.json()["message"]
    reply = ask_llm(user_msg)
    return jsonify({"reply": reply})

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=7000)

Step 6 — Run the Web App

Terminal 2 — Start Flask
cd ~/chatbot
python3 -m venv venv
source venv/bin/activate
pip install flask requests
python app.py

# Open browser: http://<UNO-Q-IP>:7000
Open any browser on the same network and navigate to the UNO Q's IP address on port 7000. You will see the chat interface and can start asking questions — all processing happens locally on the device.

Conclusion

You have built a complete offline AI chatbot that runs entirely on the Arduino UNO Q. This project teaches you how LLM inference works, how REST APIs connect frontend and backend, and how to build edge AI infrastructure without cloud dependency.
