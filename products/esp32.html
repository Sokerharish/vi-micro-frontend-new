MR Robo ESP32 — ROS2 Based Autonomous Robot

The MR Robo ESP32 brings ROS2 to microcontrollers via micro-ROS. The ESP32 runs micro-ROS firmware and interfaces with motor drivers, IMU, and proximity sensors — communicating with a full ROS2 host (Jetson / PC) over USB Serial XRCE-DDS protocol. The result is a low-cost, capable autonomous robot platform built for real embedded robotics development.

Robot Operating System (ROS)

The Robot Operating System (ROS) is an open-source robotics middleware framework that provides a collection of software libraries and tools for developing robotic applications. Although ROS is called an "operating system," it is not a traditional operating system. Instead, it acts as a software framework that simplifies the development, integration, and management of complex robotic systems.

ROS provides essential functionalities such as hardware abstraction, low-level device control, package management, inter-process communication, and implementation of commonly used robotic algorithms. It follows a modular architecture in which individual processes, called nodes, communicate with each other through message passing. These nodes can perform tasks such as sensor data acquisition, motion control, perception, planning, mapping, and actuator control.

One of the major strengths of ROS is its extensive ecosystem of reusable software packages. These include hardware drivers, robot models, simulation environments, path-planning algorithms, perception systems, and Simultaneous Localization and Mapping (SLAM) solutions. ROS also includes powerful development and debugging tools, making it widely used in robotics research and prototyping.

ROS is mainly designed for Unix-like operating systems, especially Ubuntu Linux, which is officially supported. While ROS provides flexibility and modularity, the original ROS architecture has limitations in areas such as real-time communication, scalability, reliability, and security.

System Architecture

ESP32 (micro-ROS)  →  USB Serial  →  micro-ROS Agent  →  ROS2 Host (Jetson / PC)

What is micro-ROS?

micro-ROS is a lightweight version of ROS2 designed for microcontrollers. Unlike standard ROS2 that runs on Linux-capable SBCs, micro-ROS lets the ESP32 participate directly in a ROS2 network — publishing topics, subscribing to commands, and interfacing with hardware peripherals in real time.

micro-ROS is a lightweight version of ROS 2 designed specifically for microcontrollers and resource-constrained embedded systems. It extends the capabilities of ROS 2 to low-power devices such as ESP32, STM32, RP2040, and other microcontroller platforms, enabling them to participate directly in a ROS 2 robotic network.

Unlike a standard ROS 2 system that typically runs on high-performance computers like PCs or Raspberry Pi boards, micro-ROS allows small embedded controllers to function as intelligent ROS 2 nodes. This makes it possible to perform real-time hardware interfacing, sensor acquisition, actuator control, and low-level robotic operations directly on microcontrollers.

micro-ROS on ESP32 — System Details

In this system, the ESP32 acts as the main embedded controller running micro-ROS firmware. It interfaces with multiple hardware peripherals including motor drivers, sensors, and battery monitoring circuits while communicating with the main ROS 2 ecosystem through USB serial.

The ESP32 is connected to a BTS7960 high-current motor driver module for controlling DC motors. Using PWM signals generated by the ESP32, the system can control motor direction, speed, and braking functions efficiently. The motor commands received from the ROS 2 network are processed by the ESP32 and translated into real-time motor control signals for robotic movement.

An MPU6050 inertial measurement unit (IMU) is connected through the I2C interface to provide acceleration and gyroscope data. This sensor enables orientation tracking, motion detection, tilt estimation, and stabilization functionalities for the robot. The ESP32 continuously reads IMU data and publishes it as ROS 2 topics through micro-ROS.

The system also includes battery voltage monitoring and State of Charge (SoC) estimation. The ESP32 reads the battery voltage using its internal Analog-to-Digital Converter (ADC) through a voltage divider circuit. Based on the measured voltage and battery characteristics, the controller estimates the remaining battery percentage and publishes the information to ROS 2 for power management and monitoring.

Additionally, two ultrasonic sensors are integrated for obstacle detection and distance measurement. These sensors help the robot detect nearby objects and avoid collisions during navigation. The ESP32 processes the distance measurements and sends the sensor data to the ROS 2 network in real time.

micro-ROS uses the Micro XRCE-DDS middleware to communicate with a micro-ROS Agent running on a more powerful system such as a Raspberry Pi, Jetson, or PC. The agent acts as a bridge between the embedded microcontroller and the full ROS 2 environment, allowing seamless communication between microcontroller nodes and higher-level robotic applications.

Distributed System Architecture

The complete system architecture enables distributed robotics development where:

  • ESP32 handles low-level real-time hardware control
  • micro-ROS manages ROS 2 communication on the microcontroller
  • ROS 2 on a Jetson performs advanced processing such as SLAM, navigation, AI inference, and visualization

This architecture is widely used in modern robotics applications where embedded controllers perform hardware-level tasks while higher-level computers handle computationally intensive operations such as autonomous navigation, machine learning, and decision-making.
